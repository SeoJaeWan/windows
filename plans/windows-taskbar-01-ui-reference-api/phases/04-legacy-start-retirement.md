# Phase 4. 이전 시작 조합 정리

> 이 문서는 실행용 상세 계약이다. `plan.md`의 같은 phase 요약을 기술적으로 확장하되 범위와 완료 조건은 바꾸지 않는다.

- owner_agent: `frontend-developer`
- 목적: first-party consumer가 새 `Taskbar` contract로 넘어간 뒤 old start-specific public surface와 관련 test를 지워 taskbar family의 공식 export를 하나로 정리한다.
- boundary:
  - `packages/ui/src/index.ts`
  - `packages/ui/src/index.test.ts`
  - `packages/ui/src/interactive/index.ts`
  - `packages/ui/src/components/taskbar/taskbarStartButton/**`
  - `packages/ui/src/components/taskbar/taskbarStartPanel/**`
  - `packages/ui/src/components/taskbar/taskbarSearchPanel/taskbarSearchPanel.test.tsx`
- input:
  - Phase 3 이후 `/sandbox/taskbar`는 `Taskbar` data props만 쓰는 first-party consumer stage로 동작한다.
  - 현재 `packages/ui/src/index.test.ts`는 이미 `TaskbarStartButton`, `TaskbarStartPanel`이 root export에서 사라지길 기대하지만 실제 `packages/ui/src/index.ts`와 source tree는 아직 그 상태에 도달하지 못했다.
  - `taskbarSearchPanel.test.tsx`는 surviving public surface를 검증하지만 일부 설명 문구는 legacy start-panel grammar를 끌고 있다.
- output:
  - 공개 계약:
    - `@windows/ui` root export는 `Taskbar`, `TaskbarSearch`, `TaskbarIconButton`, `TaskbarClock`, `TaskbarSearchPanel`, `TaskbarHoverPanel`, `TaskbarContextMenu`만 노출한다.
    - `TaskbarStartButton`과 `TaskbarStartPanel` source/test는 source tree에서 제거되며 legacy alias를 남기지 않는다.
    - `packages/ui/src/interactive/index.ts`는 계속 client-only placeholder로 남고 taskbar helper나 alternate assembly contract를 노출하지 않는다.
  - 내부 기본값:
    - package-private windows panel, `taskbar/common`, `utils/taskbar`는 internal implementation detail로 남는다.
    - surviving public tests는 새 taskbar family wording만 사용하고 start-specific public semantics를 다시 열지 않는다.
  - 허용하지 않는 대안:
    - `TaskbarStart*` legacy export 또는 temporary compatibility alias 유지
    - `taskbarWindowsPanel`, `taskbar/common`, `utils/taskbar`를 root export나 `@windows/ui/interactive`로 노출하는 구조
    - index-level export inventory를 internal helper snapshot으로 확장하는 구조
- 선행조건: Phase 3 완료
- 제약:
  - 이 단계는 public export 정리와 source-tree test alignment만 다루며 Storybook, additional owner surface, browser-only contract를 추가하지 않는다.
  - legacy start directory가 완전히 삭제되지 못하더라도 public import path로는 남길 수 없다.
- failure/validation: root export나 first-party consumer가 여전히 `TaskbarStart*`를 바라보거나 surviving test 서술이 start-specific public contract를 계속 frozen truth로 남기면 이 단계는 실패다.
- 작업:
  - `packages/ui/src/index.ts`와 `packages/ui/src/index.test.ts`를 surviving taskbar family 기준으로 맞춘다.
  - legacy start-specific component/test 디렉터리를 제거하거나 public source tree에서 retire한다.
  - `taskbarSearchPanel.test.tsx` 같은 surviving public boundary test의 wording drift를 새 public contract에 맞게 정리한다.
  - `interactive/index.ts`가 새 우회 export path가 되지 않도록 placeholder 상태를 유지한다.
