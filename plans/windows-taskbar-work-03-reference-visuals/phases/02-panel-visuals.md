# Phase 2. 패널과 보조 메뉴 화면 옮기기

> 이 문서는 실행 계약이다. `plan.md`의 같은 phase 요약을 기술적으로 확장하되, 범위와 완료 조건을 바꾸지 않는다.

- owner_agent: `frontend-developer`
- 목적: windows/search panel과 hover/context menu까지 같은 계열의 인상을 쓰게 해 taskbar family 전체를 한 세트처럼 보이게 한다.
- boundary:
  - `packages/ui/src/components/taskbar/taskbarWindowsPanel/**`
  - `packages/ui/src/components/taskbar/taskbarSearchPanel/**`
  - `packages/ui/src/components/taskbar/taskbarHoverPanel/**`
  - `packages/ui/src/components/taskbar/taskbarContextMenu/**`
  - `packages/ui/src/components/taskbar/internal/contentRow/**`
  - `packages/ui/src/components/taskbar/internal/panelTile/**`
- input:
  - `plans/windows-taskbar-work-03-reference-visuals/phases/01-bar-visuals.md`의 taskbar token과 bar/leaf 기본 인상
  - `https://seojaewan.com`과 `dev/blog`의 작업 시점 panel, hover, context menu 인상
  - `plans/windows-taskbar-work-02-composite-assembly/phases/01-package-assembly.md`의 package-owned windows/search surface 경계
- output:
  - 공개 계약:
    - `TaskbarWindowsPanel`, `TaskbarSearchPanel`, `TaskbarHoverPanel`, `TaskbarContextMenu`는 package 내부 기본 스타일만으로 같은 taskbar 계열의 패널과 메뉴 인상을 렌더링한다.
    - panel row, detail block, action button, menu row는 route-local card wrapper 없이도 package 내부만으로 기본 배경, 구분선, 간격, 강조 상태를 갖는다.
    - 이 phase는 패널 시각 책임을 package로 모으는 작업이며, 소비자용 새 selector나 class naming contract를 추가하지 않는다.
  - 내부 기본값:
    - 썸네일, 행 간격, 구역 배경과 구분선, 동작 강조는 package 내부 row/tile helper와 class grammar가 source of truth다.
    - 구현 중 선택한 관찰 anchor(class, token, data-attribute)는 package 내부 검증용일 뿐 public API로 승격하지 않는다.
    - 후속 Storybook sandbox는 package 기본 스타일이 이미 잡혀 있다는 계약만 소비한다.
    - 참조 사이트와 블로그는 작업 시점 참고 자료이며, 이후 자동 동기화 대상은 아니다.
  - 허용하지 않는 대안:
    - `TaskbarStartPanel`을 새 windows panel owner로 되살리는 구조
    - app/web preview wrapper가 없으면 panel chrome이 성립하지 않는 구조
    - panel visual polish를 위해 package contract 밖 임시 prop을 여는 구조
- 선행조건: `plans/windows-taskbar-work-03-reference-visuals/phases/01-bar-visuals.md` 완료
- 제약:
  - outside click, open/close runtime, persisted query state는 범위 밖이다.
  - route-local wallpaper, full-screen layout, app/web metadata는 범위 밖이다.
  - 외부 reference의 exact class 이름, pixel-perfect 복제, 이후 live sync는 범위 밖이다.
- failure/validation: package가 아니라 story/app wrapper나 임시 local CSS가 패널과 메뉴 인상의 대부분을 책임지게 되면 실패다.
- 작업:
  - windows/search panel, hover/context menu, row/tile helper의 배경, 구분, 간격, 강조 규칙을 같은 token 체계 안으로 정리한다.
  - section, row, detail, action의 시각 책임을 package component 내부로 이동한다.
  - 후속 Storybook sandbox가 package 기본 스타일만으로 화면을 보여줄 수 있도록 panel/menu의 시각 ownership을 package 안에 남긴다.
