# Phase 1. Hover 미리보기 만들기

> 이 문서는 실행용 상세 계약이다. `plan.md`의 같은 phase 요약을 기술적으로 확장하되, 범위나 결론을 새로 바꾸지 않는다.

- owner_agent: `frontend-developer`
- 목적: `TaskbarHoverPanel`을 icon-attached preview surface로 고정해 vertical menu와 다른 compact preview 역할을 분리한다.
- boundary:
  - `packages/ui/src/index.ts`
  - `packages/ui/src/components/taskbar/taskbarHoverPanel/**`
  - `packages/ui/src/components/taskbar/internal/**`
  - `plans/windows-taskbar-04-attached-surfaces/reference-captures.md`
- input:
  - 선행 계약:
    - `packages/ui/src/index.ts`와 `packages/ui/src/components/taskbar/**`가 다음 canonical public taskbar contract를 실제 파일 기준으로 제공할 것: `Taskbar`, `TaskbarWindowsButton`, `TaskbarSearch`, `TaskbarIconButton`, `TaskbarClock`, `TaskbarWindowsPanel`, `TaskbarSearchPanel`, `TaskbarHoverPanel`, `TaskbarIconContextMenu`; 그리고 `packages/ui/src/components/taskbar/internal/**`에 taskbar 전용 private primitive가 존재할 것.
  - 기준 캡처:
    - `taskbar-hover-preview.png`
  - 현재 실행 계약:
    - `packages/ui/package.json`의 `test` 스크립트는 `vitest run`
    - TypeScript 검증은 `pnpm --filter @windows/ui exec tsc --noEmit -p tsconfig.json`
- output:
  - 공개 계약:
    - `TaskbarHoverPanel`은 public attached surface이고, app label + preview card strip 구조를 가진다.
    - preview surface는 menu가 아니라 compact preview처럼 읽혀야 한다.
    - panel root는 자기 크기, padding, background, border, shadow만 소유하고 anchor positioning은 소유하지 않는다.
  - 내부 기본값:
    - `TaskbarHoverPanel`은 `PanelSurface`, `PreviewCard`, `SectionHeading` 계열 private primitive를 조합한다.
    - preview card는 label보다 시각적 무게가 더 크고, 전체 인상은 좁고 낮은 floating strip에 가깝다.
  - 허용하지 않는 대안:
    - hover preview를 세로 action list 메뉴처럼 만드는 구조
    - `TaskbarHoverPanel`을 `TaskbarSearchPanel`이나 `TaskbarWindowsPanel` 내부 sub-block으로 취급하는 구조
- 선행조건:
  - `plans/windows-taskbar-01-foundation-shell/phases/01-public-contract-and-primitives.md`의 output이 현재 소스 트리에 반영되어 있을 것
  - `packages/ui/src/index.ts`와 `packages/ui/src/components/taskbar/**`가 다음 canonical public taskbar contract를 실제 파일 기준으로 제공할 것: `Taskbar`, `TaskbarWindowsButton`, `TaskbarSearch`, `TaskbarIconButton`, `TaskbarClock`, `TaskbarWindowsPanel`, `TaskbarSearchPanel`, `TaskbarHoverPanel`, `TaskbarIconContextMenu`; 그리고 `packages/ui/src/components/taskbar/internal/**`에 taskbar 전용 private primitive가 존재할 것.
- 제약:
  - 이 단계는 hover delay, close logic, thumbnail live preview를 다루지 않는다.
- failure/validation:
  - hover preview가 compact preview가 아니라 menu처럼 보이면 실패다.
  - positioning 책임이 public contract로 새어 나오면 실패다.
- 작업:
  - app label과 preview card strip을 가진 `TaskbarHoverPanel` layout을 정리한다.
  - preview card 비율, padding, floating surface density를 캡처 기준으로 맞춘다.
  - menu 계열과 섞이지 않게 preview 전용 hierarchy를 유지한다.
