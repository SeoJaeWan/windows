# Phase 2. Search 패널 내부 메뉴 고정

> 이 문서는 실행용 상세 계약이다. `plan.md`의 같은 phase 요약을 기술적으로 확장하되, 범위나 결론을 새로 바꾸지 않는다.

- owner_agent: `frontend-developer`
- 목적: 검색 결과 행의 context surface를 Search panel 내부 secondary surface로 고정해 taskbar icon menu와 책임을 분리한다.
- boundary:
  - `packages/ui/src/components/taskbar/taskbarSearchPanel/**`
  - `packages/ui/src/components/taskbar/internal/**`
  - `plans/windows-taskbar-03-search-panel/reference-captures.md`
- input:
  - Phase 1 output:
    - `TaskbarSearchPanel`이 `default`, `query-results`, `query-detail`, `query-empty` 네 primary state를 정적 UI로 구분해 보여 준다.
  - 기준 캡처:
    - `search-result-context-menu.png`
    - `search-panel-query-detail.png`
    - `search-panel-query-detail-pinned.png`
  - 현재 실행 계약:
    - `packages/ui/package.json`의 `test` 스크립트는 `vitest run`
    - TypeScript 검증은 `pnpm --filter @windows/ui exec tsc --noEmit -p tsconfig.json`
- output:
  - 공개 계약:
    - `TaskbarSearchPanel`은 결과 행 context surface를 statically 표현할 수 있어야 하지만, 이를 위해 별도 public `TaskbarContextMenu`를 만들지 않는다.
    - search result row secondary surface는 `파일 실행`, `파일 위치 열기`, `시작 화면에 고정`, `작업 표시줄에서 제거` action list를 가진 compact menu다.
    - row context surface는 `TaskbarSearchPanel` 내부 scenario로 소유하고 `TaskbarIconContextMenu`와 이름이나 책임을 공유하지 않는다.
  - 내부 기본값:
    - row context surface는 `ActionMenuList`, `ListRow`, `Icon` 계열 private primitive를 사용한다.
    - `파일 실행`, `파일 위치 열기`, pin/unpin, taskbar remove glyph는 Fluent action icon 후보를 쓸 수 있다.
    - primary state와 row context surface를 동시에 보여 줄 때도 detail pane hierarchy가 무너지지 않게 spacing을 유지한다.
  - 허용하지 않는 대안:
    - search result row menu를 `TaskbarIconContextMenu`에 합치는 구조
    - detail pane action list와 row context menu를 같은 body로 축소하는 구조
    - row context surface를 새 public export로 노출하는 구조
- 선행조건:
  - `plans/windows-taskbar-03-search-panel/phases/01-results-and-detail.md`의 output이 현재 소스 트리에 반영되어 있을 것
- 제약:
  - 실제 우클릭 트리거와 위치 anchor는 여전히 다루지 않는다.
  - `java` 같은 query별 데이터 차이는 실제 검색 엔진이 아니라 static visual density 기준으로만 반영한다.
- failure/validation:
  - search row menu가 taskbar icon menu와 같은 구조로 오인되면 실패다.
  - row context surface를 public generic menu로 다시 끌어올리면 실패다.
- 작업:
  - result row context menu의 action order와 compact density를 캡처 기준으로 정리한다.
  - detail pane action list와 row context menu의 역할 차이가 드러나게 icon, spacing, hierarchy를 조정한다.
  - row context surface를 Search panel 내부 scenario로 표현 가능하게 만든다.
- 검증:
  - [ ] `pnpm --filter @windows/ui test`
  - [ ] `pnpm --filter @windows/ui exec tsc --noEmit -p tsconfig.json`
  - [ ] Search result row context surface가 `파일 실행`, `파일 위치 열기`, `시작 화면에 고정`, `작업 표시줄에서 제거`를 가진 compact menu로 보인다.
  - [ ] row context surface 책임이 `TaskbarIconContextMenu`나 generic public menu로 새어 나가지 않는다.
  - [ ] detail pane action list와 row context menu가 서로 다른 secondary surface로 읽힌다.
