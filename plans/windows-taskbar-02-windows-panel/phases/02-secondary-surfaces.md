# Phase 2. Windows 패널 내부 메뉴 고정

> 이 문서는 실행용 상세 계약이다. `plan.md`의 같은 phase 요약을 기술적으로 확장하되, 범위나 결론을 새로 바꾸지 않는다.

- owner_agent: `frontend-developer`
- 목적: `TaskbarWindowsPanel` 내부 secondary surface를 고정해 all-item menu와 pinned-item menu의 차이를 panel owner 안에서 명확히 분리한다.
- boundary:
  - `packages/ui/src/components/taskbar/taskbarWindowsPanel/**`
  - `packages/ui/src/components/taskbar/internal/**`
  - `plans/windows-taskbar-02-windows-panel/reference-captures.md`
- input:
  - Phase 1 output:
    - `TaskbarWindowsPanel`이 `pinned`, `all`, `query-empty` 세 primary mode를 정적 UI로 구분해 보여 준다.
  - 기준 캡처:
    - `start-all-item-context-menu.png`
    - `start-pinned-context-menu-right-only.png`
    - `start-pinned-context-menu-left-right.png`
    - `start-pinned-context-menu-forward-left-right.png`
    - `start-pinned-context-menu-forward-left.png`
  - 현재 실행 계약:
    - `packages/ui/package.json`의 `test` 스크립트는 `vitest run`
    - TypeScript 검증은 `pnpm --filter @windows/ui exec tsc --noEmit -p tsconfig.json`
- output:
  - 공개 계약:
    - `TaskbarWindowsPanel`은 item-level secondary surface를 statically 표현할 수 있어야 하지만, 이를 위한 generic public `TaskbarContextMenu`를 다시 만들지 않는다.
    - `all-item` secondary surface는 `파일 위치 열기`, `시작 화면에 고정`, `작업 표시줄에서 제거` action list를 가진 compact menu다.
    - `pinned-item` secondary surface는 `파일 위치 열기`, `시작 화면에서 제거`, `작업 표시줄에서 제거`에 더해 tile 위치에 따라 move action subset이 달라진다.
    - `pinned-item` move action subset은 관찰 기준으로 `right-only`, `left-right`, `forward-left-right`, `forward-left` 네 패턴을 가진다.
    - secondary surface는 `TaskbarWindowsPanel` 내부 scenario로 소유하고 별도 public export를 만들지 않는다.
  - 내부 기본값:
    - menu row는 `ActionMenuList`, `ListRow`, `Icon`, `MenuDivider` 계열 private primitive를 조합한다.
    - `작업 표시줄에서 제거`, `시작 화면에 고정`, 이동 관련 glyph는 Fluent action icon 후보를 사용할 수 있다.
    - secondary surface 시각 상태는 presentational scenario selection으로만 표현하고 event API는 추가하지 않는다.
  - 허용하지 않는 대안:
    - Windows panel 내부 메뉴를 `TaskbarIconContextMenu`와 같은 public owner로 합치는 구조
    - pinned tile 위치별 move action 차이를 무시하고 하나의 고정 메뉴로 축소하는 구조
    - `all-item` menu와 `pinned-item` menu를 같은 action list 구조로 축소하는 구조
- 선행조건:
  - `plans/windows-taskbar-02-windows-panel/phases/01-panel-modes.md`의 output이 현재 소스 트리에 반영되어 있을 것
- 제약:
  - 이 단계도 여전히 정적 UI만 다룬다.
  - 실제 우클릭 이벤트와 위치 anchor는 소유하지 않는다.
- failure/validation:
  - secondary surface가 panel 외부 generic context menu owner를 다시 요구하면 실패다.
  - 4개 pinned menu 패턴 중 하나라도 누락돼 위치 기반 차이가 사라지면 실패다.
- 작업:
  - `all-item` menu row와 action order를 캡처 기준으로 정리한다.
  - `pinned-item` menu의 공통 action과 위치별 move action subset을 정리한다.
  - `right-only`, `left-right`, `forward-left-right`, `forward-left` 네 variant를 static scenario로 표현 가능하게 만든다.
  - Windows panel의 primary mode와 secondary surface가 함께 렌더링돼도 layout ownership이 흐려지지 않게 spacing을 조정한다.
- 검증:
  - [ ] `pnpm --filter @windows/ui test`
  - [ ] `pnpm --filter @windows/ui exec tsc --noEmit -p tsconfig.json`
  - [ ] `TaskbarWindowsPanel` 내부에 `all-item` menu와 `pinned-item` menu가 구분된 secondary surface로 존재한다.
  - [ ] `pinned-item` menu가 `right-only`, `left-right`, `forward-left-right`, `forward-left` 네 move-action 패턴을 표현한다.
  - [ ] `TaskbarIconContextMenu`나 generic `TaskbarContextMenu`에 Windows panel 메뉴 책임이 새로 흘러가지 않는다.
