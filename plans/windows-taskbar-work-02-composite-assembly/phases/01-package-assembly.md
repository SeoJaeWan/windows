# Phase 1. package 안에서 taskbar 조립하기

> 이 문서는 실행 계약이다. `plan.md`의 같은 phase 요약을 기술적으로 확장하되, 범위와 완료 조건을 바꾸지 않는다.

- owner_agent: `frontend-developer`
- 목적: `Taskbar`가 helper plain model을 직접 소비해 icon strip, windows surface, search surface, clock 영역을 package 내부에서 조합하게 한다.
- boundary:
  - `packages/ui/src/components/taskbar/taskbar/**`
  - `packages/ui/src/components/taskbar/taskbarSearchPanel/**`
  - `packages/ui/src/components/taskbar/taskbarWindowsPanel/**`
  - `packages/ui/src/components/taskbar/taskbarIconButton/**`
  - `packages/ui/src/components/taskbar/taskbarClock/**`
- input:
  - `plans/windows-taskbar-work-01-data-helper-foundation/phases/02-helper-projection.md`의 helper contract와 plain output model
  - 현재 `packages/ui/src/components/taskbar/taskbar/index.tsx`는 `startButton`, `search`, `items`, `clock` raw slot 조합을 받는다.
- output:
  - 공개 계약:
    - `Taskbar`는 `entries`, `icons`, `windows`, `search`, `clock`만으로 icon strip, windows surface, search surface, clock block을 package 내부에서 직접 조합한다.
    - 소비자는 `Taskbar`를 쓰기 위해 leaf ReactNode를 직접 조합하지 않는다.
    - `Taskbar`는 후속 visuals와 sandbox가 재사용할 `taskbar`, `data-panel="windows"`, `data-panel="search"`, `data-mode="pinned" | "all" | "default" | "results"` anchor를 package source에서 직접 만든다.
  - 내부 기본값:
    - icon strip은 helper가 계산한 reserved launcher와 entry-linked icon model을 사용한다.
    - windows/search panel은 helper가 계산한 discriminated model을 사용한다.
    - `TaskbarStartButton`, `TaskbarStartPanel`은 이 phase에서 조합 owner가 아니며, 공개 export 제거는 다음 phase가 맡는다.
  - 허용하지 않는 대안:
    - `Taskbar`가 helper 대신 route fixture나 story fixture shape에 맞춰 조합 규칙을 다시 계산하는 구조
    - raw slot과 data-driven composite가 둘 다 공식 조합 경로로 남는 구조
    - package-owned windows/search panel 대신 legacy start panel이 조합 owner처럼 되살아나는 구조
- 선행조건: `plans/windows-taskbar-work-01-data-helper-foundation/phases/02-helper-projection.md` 완료
- 제약:
  - 이 phase는 package 조합 ownership만 다룬다. root export 제거, web sandbox consumer 정리, glass style, Storybook, app/web retirement는 범위 밖이다.
  - hover/context registry projection은 필수 산출물이 아니다.
- failure/validation: `Taskbar`가 여전히 leaf ReactNode 조합을 요구하거나, DOM anchor가 package 밖에서만 만들어지면 실패다. root export test나 web sandbox가 아직 옛 계약을 쓴다는 이유로 package boundary를 다시 넓히면 범위 이탈이다.
- 작업:
  - `Taskbar` root가 helper output을 받아 icon strip, windows/search surface, clock를 직접 조합하도록 바꾼다.
  - package-owned windows/search panel 경계를 composite 안에서 사용하도록 연결한다.
  - 후속 visual/sandbox plan이 재사용할 DOM anchor를 composite source 기준으로 고정한다.
- 검증:
  - [ ] `pnpm --filter @windows/ui exec vitest run src/components/taskbar/taskbar/taskbar.test.tsx`
  - [ ] `pnpm --filter @windows/ui exec tsc --noEmit -p tsconfig.json`
  - [ ] `packages/ui/src/components/taskbar/taskbar/taskbar.test.tsx`가 package-owned composite surface를 공식 owner로 설명한다.
