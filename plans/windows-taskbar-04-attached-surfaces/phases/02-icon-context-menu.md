# Phase 2. 작업 아이콘 메뉴 만들기

> 이 문서는 실행용 상세 계약이다. `plan.md`의 같은 phase 요약을 기술적으로 확장하되, 범위나 결론을 새로 바꾸지 않는다.

- owner_agent: `frontend-developer`
- 목적: root taskbar icon 우클릭 menu를 `TaskbarIconContextMenu`라는 좁은 public owner로 고정해 panel 내부 menu와 책임을 분리한다.
- boundary:
  - `packages/ui/src/index.ts`
  - `packages/ui/src/components/taskbar/taskbarIconContextMenu/**`
  - `packages/ui/src/components/taskbar/internal/**`
  - `plans/windows-taskbar-04-attached-surfaces/reference-captures.md`
- input:
  - Phase 1 output:
    - `TaskbarHoverPanel`이 menu가 아니라 compact preview surface로 읽힌다.
  - 기준 캡처:
    - `taskbar-icon-context-menu.png`
  - 현재 실행 계약:
    - `packages/ui/package.json`의 `test` 스크립트는 `vitest run`
    - TypeScript 검증은 `pnpm --filter @windows/ui exec tsc --noEmit -p tsconfig.json`
- output:
  - 공개 계약:
    - `TaskbarIconContextMenu`는 root taskbar icon 우클릭 menu만 소유하는 public component다.
    - menu는 제목 `작업`, recent item 세로 목록, 구분선, 하단 action row 두 개 구조를 가진다.
    - `TaskbarIconContextMenu`는 Windows panel menu나 Search result menu를 소유하지 않는다.
    - recent item의 2차 우클릭 메뉴처럼 아직 안정적으로 재현하지 못한 nested menu는 이번 canonical contract에 포함하지 않는다.
  - 내부 기본값:
    - `TaskbarIconContextMenu`는 `PanelSurface`, `ListRow`, `ActionMenuList`, `MenuDivider`, `Icon` 계열 private primitive를 조합한다.
    - action glyph는 Fluent action icon 후보를 사용할 수 있다.
    - recent item 목록과 하단 action row는 같은 menu 안에 있어도 hierarchy가 분명해야 한다.
  - 허용하지 않는 대안:
    - `TaskbarIconContextMenu`를 다시 generic `TaskbarContextMenu`로 넓히는 구조
    - root icon menu 책임에 Search result menu나 Windows panel item menu를 같이 섞는 구조
    - 재현하지 못한 nested recent-item context menu를 canonical contract에 넣는 구조
- 선행조건:
  - `plans/windows-taskbar-04-attached-surfaces/phases/01-hover-preview.md`의 output이 현재 소스 트리에 반영되어 있을 것
- 제약:
  - 실제 right-click trigger, nested recent interaction, anchor positioning은 다루지 않는다.
- failure/validation:
  - root icon menu와 search/windows 내부 menu 경계가 무너지면 실패다.
  - 재현하지 못한 nested recent menu가 이번 phase 구현 의무처럼 들어오면 실패다.
- 작업:
  - 제목, recent item list, divider, bottom action row를 가진 `TaskbarIconContextMenu` 구조를 정리한다.
  - recent item row와 bottom action row의 hierarchy와 spacing을 조정한다.
  - `TaskbarIconContextMenu`의 책임이 root icon menu에만 머물도록 naming과 문서, 파일 경계를 정리한다.
