# Phase 1. Taskbar 데이터 입력 확정

> 이 문서는 실행용 상세 계약이다. `plan.md`의 같은 phase 요약을 기술적으로 확장하되 범위와 완료 조건은 바꾸지 않는다.

- owner_agent: `frontend-developer`
- 목적: `Taskbar`가 받을 유일한 public data contract와 `utils/taskbar`가 반환할 유일한 plain output model을 먼저 닫아 이후 구현과 test materialization이 같은 기준을 사용하게 한다.
- boundary:
  - `packages/ui/src/components/taskbar/taskbar/**`
  - `packages/ui/src/components/taskbar/taskbarSearchPanel/**`
  - `packages/ui/src/components/taskbar/taskbarStartPanel/**`
  - `packages/ui/src/components/taskbar/taskbarStartButton/**`
  - `packages/ui/src/utils/taskbar/**`
  - `packages/ui/src/index.ts`
- input:
  - 현재 `packages/ui/src/components/taskbar/taskbar/index.tsx`는 `startButton`, `search`, `items`, `clock` slot을 받지만 `packages/ui/src/components/taskbar/taskbar/taskbar.test.tsx`는 이미 `entries`, `icons`, `windows`, `search`, `clock` data props를 기대한다.
  - `packages/ui/src/index.test.ts`는 root export에서 `TaskbarStartButton`, `TaskbarStartPanel`이 제거되기를 기대하지만 실제 `packages/ui/src/index.ts`는 아직 legacy start export를 유지한다.
  - `apps/web/src/app/sandbox/taskbar/page.test.tsx`와 `e2e/sandbox-taskbar-preview.spec.ts`는 `/sandbox/taskbar`가 `taskbar-sandbox-canonical` / `taskbar-sandbox-compare` marker 아래 새 `Taskbar` contract를 소비하길 기대한다.
  - 실행 계약은 `packages/ui/package.json`의 `test`와 `pnpm --filter @windows/ui exec tsc --noEmit -p tsconfig.json`이다.
- output:
  - 공개 계약:
    - `Taskbar`의 top-level 입력은 `entries`, `icons`, `windows`, `search`, `clock`뿐이며 `className`과 native `nav` props만 additive override로 허용한다.
    - `entries`의 공식 shape는 `id`, `category`, `title` 필수와 `summary`, `meta`, `href`, `tags`, `thumbnailSrc`, `thumbnailAlt`, `windows`, `search` 선택 필드다.
    - `icons`의 공식 shape는 `id`, `category`, `kind`, `label` 필수와 `status` 선택 필드다.
    - `windows`의 공식 shape는 `view?: "pinned" | "all"`, `onPinnedSelect: (id: string) => void`, `onAllSelect: (id: string) => void`다.
    - `search`의 공식 shape는 `view?: "default" | "results"`, `value?: string`, `placeholder?: string`, `onResultSelect: (id: string) => void`, `onActionSelect?: (id: string) => void`다.
    - `clock`의 공식 shape는 `timeLabel: string`, `dateLabel: string`다.
    - standalone public surface는 `TaskbarSearch`, `TaskbarIconButton`, `TaskbarClock`, `TaskbarSearchPanel`, `TaskbarHoverPanel`, `TaskbarContextMenu`만 유지한다.
  - 내부 기본값:
    - `windows.view`가 없으면 `pinned`가 winner다.
    - `search.view`는 explicit 값이 우선이고 explicit 값이 없을 때 `search.value`가 비어 있지 않으면 `results`, 아니면 `default`가 winner다.
    - `entry.windows?.visible`과 `entry.search?.searchable`의 기본값은 `true`, `entry.windows?.pinned`, `entry.search?.recommended`, `entry.search?.featured`의 기본값은 `false`다.
    - `entry.windows?.order`와 `entry.search?.rank`는 숫자 오름차순 우선이고 동률과 미지정은 선언 순서 fallback이다.
    - search results 필터는 `title`, `summary`, `meta`, `tags[]`에 대한 대소문자 구분 없는 포함 매칭이다.
    - `category="windows"`만 entry 없이 허용되는 reserved launcher icon이며 다른 icon category는 매칭되는 entry가 없으면 drop한다.
    - icon helper의 공식 출력은 `Array<{ id, label, kind, status, category, entryId?: string, reservedLauncher: boolean }>`다.
    - windows helper의 공식 출력은 `mode: "pinned"`일 때 `items: Array<{ id, title, summary?: string, thumbnailSrc?: string, thumbnailAlt?: string }>`를, `mode: "all"`일 때 `sections: Array<{ id, label, items: Array<{ id, title, summary?: string, meta?: string }> }>`를 가진 discriminated model이다.
    - search helper의 공식 출력은 `mode: "default"`일 때 `recommendedItems: Array<{ id, label, meta?: string, kind?: string }>`와 `featuredItems: Array<{ id, label, description?: string, thumbnailSrc?: string, thumbnailAlt?: string }>`를 가진다.
    - search helper의 공식 출력은 `mode: "results"`일 때 `query: string`, `resultItems: Array<{ id, label, meta?: string, active?: boolean, kind?: string }>`, `detail?: { title, description?: string, metadata?: string[], actions?: Array<{ id, label }> }`를 가진다.
    - `utils/taskbar`의 공식 출력은 JSX, `ReactNode`, `string[]`, DOM prop bundle이 아니라 icon strip model, windows panel model, search panel model 같은 plain render model이다.
    - search results에서 active row와 detail winner는 정렬 후 첫 결과다. 결과가 비면 `detail`도 비우고 synthetic fallback row를 만들지 않는다.
  - 허용하지 않는 대안:
    - raw `startButton/search/items/clock` slot contract
    - public `pinnedIds`, `allIds`, `recommendedIds`, `featuredIds`
    - helper output을 `string[]`, full entry passthrough, JSX-ready prop bag 가운데 하나로 구현 시점에 다시 고르는 구조
    - `utils/taskbar` helper가 public import path나 root export로 노출되는 구조
    - `TaskbarStartButton`, `TaskbarStartPanel`을 canonical taskbar family와 병치하는 구조
- 선행조건: `none`
- 제약:
  - 이 단계에서 `apps/web`와 `e2e`는 읽기 전용 validation context이며 editable boundary가 아니다.
  - hover/context surface를 `entries` registry 기반 projection helper로 다시 설계하지 않는다. 이 plan에서 data-driven contract로 닫는 대상은 `Taskbar` composite, icon strip, windows panel, search panel이다.
- failure/validation: `utils/taskbar` output이 `id 목록`, `entry projection`, `render-ready model` 사이에서 다시 열리거나 search detail winner rule과 reserved icon rule이 helper마다 다르게 해석되면 이 단계는 완료가 아니다.
- 작업:
  - `Taskbar` public input type을 하나의 data-driven contract로 닫고 raw slot sibling contract를 명시적으로 배제한다.
  - icon strip, windows panel, search panel helper의 plain output model을 phase 문서 기준으로 고정해 `plan-materialize`가 file shape를 추측하지 않게 한다.
  - public으로 남길 leaf surface와 retire할 start-specific surface를 이 단계에서 먼저 확정한다.
