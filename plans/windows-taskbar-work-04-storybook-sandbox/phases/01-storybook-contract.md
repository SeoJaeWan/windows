# Phase 1. Storybook 실행 경로 열기

> 이 문서는 실행 계약이다. `plan.md`의 같은 phase 요약을 기술적으로 확장하되, 범위와 완료 조건을 바꾸지 않는다.

- owner_agent: `frontend-developer`
- 목적: taskbar sandbox의 공식 owner를 app/web route가 아니라 `@windows/ui` Storybook로 고정한다.
- boundary:
  - `packages/ui/package.json`
  - `packages/ui/.storybook/**`
  - `packages/ui/tsconfig.json`
  - `pnpm-lock.yaml`
- input:
  - `plans/windows-taskbar-work-03-reference-visuals/phases/02-panel-visuals.md`는 package-owned visual grammar와 package-only chrome ownership을 이미 제공한다.
  - 현재 repo에는 `@windows/ui` package-local Storybook 실행 경로가 없고, Storybook script/devDependency도 선언돼 있지 않다.
- output:
  - 공개 계약:
    - taskbar package-owned sandbox의 공식 진입점은 `pnpm --filter @windows/ui storybook`과 `pnpm --filter @windows/ui build-storybook`이다.
    - Storybook script와 devDependency의 owner는 root tooling이 아니라 `packages/ui/package.json`이다.
    - Storybook config는 `packages/ui/.storybook/**`에만 존재하고, app/web route는 prerequisite가 아니다.
  - 내부 기본값:
    - workspace install로 `pnpm-lock.yaml`이 갱신될 수 있지만, Storybook tooling ownership은 `@windows/ui`에 남는다.
    - taskbar reference surface는 `@windows/ui` source를 직접 읽고, Next runtime이나 app/web metadata에 의존하지 않는다.
    - `plans/windows-taskbar-work-05-web-sandbox-retirement/phases/01-remove-route-owner.md`는 이 Storybook contract를 replacement sandbox 선행조건으로 사용한다.
  - 허용하지 않는 상태:
    - Storybook dependency를 root `package.json`이나 별도 app package에 두고 `@windows/ui`는 story source만 소유하는 구조
    - Storybook owner surface가 `apps/web` dev server나 route-local CSS를 prerequisite로 두는 구조
    - package-local script 없이 ad hoc CLI 조합으로만 sandbox를 여는 구조
- 선행조건: `plans/windows-taskbar-work-03-reference-visuals/phases/02-panel-visuals.md`가 package-owned panel/overlay visual grammar와 package-only chrome ownership을 제공한 상태
- 제약:
  - 이 phase는 tooling 경계만 연다. 실제 taskbar reference story는 다음 phase가 맡는다.
  - browser-runner와 visual snapshot runner는 범위 밖이다.
- failure/validation: sandbox owner가 package가 아니라 app/web나 root tooling으로 다시 돌아가면 실패다.
- 작업:
  - `@windows/ui` package-local Storybook script, devDependency, config를 추가한다.
  - Storybook이 package source를 직접 읽도록 ts/config 경계를 정리한다.
  - workspace lockfile 변경이 생기더라도 Storybook tooling owner가 `packages/ui`라는 점이 문서와 파일 배치에 드러나게 한다.
  - 후속 phase가 taskbar reference story를 얹을 수 있는 최소 실행/빌드 경로를 닫는다.
- 검증:
  - [ ] `pnpm --filter @windows/ui build-storybook`
  - [ ] `pnpm --filter @windows/ui exec tsc --noEmit -p tsconfig.json`
  - [ ] `packages/ui/package.json`, `packages/ui/.storybook/**`, `pnpm-lock.yaml`만으로 Storybook dependency placement와 package-owned sandbox 실행 경로가 설명된다.
